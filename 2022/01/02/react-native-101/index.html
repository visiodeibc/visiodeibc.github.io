#Crossplatform 개발

Crossplatform (ios + android) 개발에서 선두주자를 달리고 있는 두가지 플랫폼이 있다면 바로
Flutter와 React Native이다.

구글에서 개발하고 유지하는 flutter는 네이티브 개발에 버금가는 퍼포먼스를 내면서 가장 빠른 속도로 성장하고 있는 프레임 워크이다.
dart라는 언어를 기반으로 하여모든 컴포넌트를 위젯 단위로 쪼개 쉽고 직관적으로 구성이 가능하다.
catch phrase가 'Build apps for any screens'처럼 ios, aos를 넘어 웹, 리눅스, 맥, 윈도우 어플리케이션까지의 호환을 도전하고 있다.

React Native(이하 RN)은 React프레임 워크를 개발한 페이스북에서 메인테인하고 있는 프로젝트로서 퍼포먼스면에서는 좀 떨어져도
기존 javascript를 사용하던 개발자들이 부담없이 시작할 수 있어 이 또한 선두를 달리고 있다.
catch phrase가 'learn once, write anywhere'에서 볼 수 있듯이 javascript로 사용가능한 편리함을 강조한다.

새로 오게된 회사에서는 react native를 사용해서 앱을 만들고 있다.
flutter 경험이 있어서 상대적으로 좀 쉬운 온보딩을 바라는 맘은 있지만 역시나 해봐야 알 듯하다.
오늘은 Mac환경에서 react native를 설치하고 세팅하는 기본을 할거다.

#설치

설치 및 실행하기 위해서는 'expo'라는 cli를 사용할 수도 있고 RN 네이티브 cli를 사용할 수도 있다. expo는 RN을 위한 다양한 라이브러리들을
모아둔 패키지로 실행 및 테스팅을 간편히 실행할 수 있게 해주는 것으로 보인다.
무엇이든 어렵게 하면 쉬운거는 금방 배우니 난 우선 RN 네이티브 cli로 세팅을 해보겠다.
설치 공식 document는 여기에서 확인할 수 있다.

  1. Dependencies 설치
    home brew 가 설치되어 있다는 가정 하에…

  2. virtual device 셋업

    2-1) ios

    1. ios의 경우 xcode를 설치한
    2. xcode상에서 preferences -> components 메뉴에 가서 타겟 ios 시뮬레이터를 설치한다.
    3. xcode상에서 preferences -> locations으로 가서 command line tool(하나 밖에 없을 것이다.)을 설정해준다.
    4. 터미널에서 sudo gem install cocoapods로 cocoapods설치

    2-2) aos

    1. java를 설치한다.
    2. android studio 를 설치한다. - 설치시 다음 요소들이 체크 되어 있는지 확인한다. -> Android SDK, Android SDK Platform, Android Virtual Device
    3. 초기 세팅에서 More Action -> SDK Manager 에서 Android 10 (Q)를 추가 설치해준다.
    4. $HOME/.bash_profile 혹 $HOME/.bashrc 혹 ~/.zprofile 혹 ~/.zshrc 에 들어가 아래 부분을 추가한다.
    5. 터미널에서 나래 커멘드를 기입한다

  3. 프로젝트 생성
    원하는 디렉토리로 이동 한 후에 프로젝트를 생성한다.

  4. 앱 실행

    1. 생성 프로젝트 안으로 터미널 통해 이동
    2. npx react-native start를 기입 후 실행
    3. 새로운 터미널 시작
    4. ios의 경우 npx react-native run-ios을 기입 aos의 경우 npx react-native run-android를 실행
    5. 위 단계를 모두 실행하면 아래와 같이 시뮬레이터가 실행 된 후에 기본 앱이 나올 것이다.

끝!!!
이상 react native 설치 방법!
