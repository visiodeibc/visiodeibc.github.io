regex 101

#RegEx?

RegEx는 regular expression의 약자로 text, file을 서치할때 효율적이고 빠르게 찾을 수 있도록 도움이 되는 친구다.
나의 하늘같은 사수님께서 sql 쿼리와 regex는 배워두면 두고두고 써먹을때가 있다고 하셔서 해야하는데… 생각만하다
쉽게 가르쳐 주고 알려주는 사이트를 발견하여 이참에 배워두려고 한다.

#Let’s 기릿!

#Basic Matcher

  1. . 는 모든 캐릭터에 매칭이 된다.
  2. 케릭터 사이에 []를 사용해주면 해당 케릭터와 []안에 있는 모든 케릭터에 매칭이 된다. ex)b[ae]r -> bar, ber
  3. 케릭터 사이에 [^]를 사용할때 ^뒤에 케릭터를 추가하면 해당 케릭터를 제외한 모든 텍스트가 매칭된다. ex) b[^a]r -> 매치: ber, bur 비매치: bar
  4. [x-y] x~y사이에 있는 케릭터가 매치 된다. (숫자, 알파벳 공통)

#Repetitions

  1. 케릭터 뒤에 * 가 붙으면 바로 앞 케릭터가 없거나 n개 있는 경우까지 매칭된다. ex)be*r -> br, ber, beer
  2. 케릭터 뒤에 + 가 붙으면 바로 앞 케릭터가 1 ~ n개 있는 경우까지 매칭된다. ex)be+r -> ber, beer
  3. 케릭터 뒤에 ? 가 붙으면 바로 앞 케릭터가 있거나 없는경우 매칭된다. ex)be?r -> br, ber
  4. {x}가 있는 경우 x만큼의 숫자만큼 케릭터가 있는 경우가 매칭된다. ex)be{2}r -> beeer
  5. {x,}가 있는 경우 x 숫자 이상의 케릭터가 있는 경우가 매칭된다. ex)be{2,}r -> beer, beeer
  6. {x,y}가 있는 경우 x, y중간만큼 케릭터가 있는 경우가 매칭된다. ex)be{1,2}r -> ber, beer

#Grouping

  1. () 를 사용할 경우 grouping이 되는데 이 사이에 있는 케릭터가 그루핑 된다.
  2. (ha)-\1,(haa)-\2 를 활용하여 referencing이 된다는데 이해를 못했다…
  3. |는 or의 표현이다 앞에 있거나 뒤에 있는 표현을 매칭해준다. ex) (C|c)at -> Cat, cat
  4. { } [ ] / \ + * . $^ | ?들은 regex에서 사용하는 예약어 이다. 이 문자를 고르려면 그 앞에 \를 추가하여야 한다. ex) (\*|\.) -> *, .
  5. ^의 경우 해당 라인이 ^뒤에 오는 캐릭터로 시작하는 line을 찾아준다.
  6. $의 경우 해당 라인이 $앞에 나오는 케릭터로 끝나는 부분을 찾아준다.
  7. \w는 레터, 숫자, 언더스코어를 찾아준다. ex) abcABC123 _
  8. \W는 레터, 숫자, 언더스코어를 제외하고 찾아준다. ex) .:!?
  9. \d는 숫자를 찾아준다.
  10. \D는 숫자를 제외하고 찾아준다.
  11. \s는 스페이스를 찾아준다.
  12. \S는 스페이스를 제외하고 찾아준다.

#Lookaround

  1. (?=)는 =뒤에 오는 케릭터와 매치되는 케릭터를 찾아준다. ex) \d+(?=PM) -> 3PM
  2. (?!)는 !뒤에 오는 케릭터와 없는 경우 케릭터를 찾아준다. ex) \d+(?!PM) -> 매치: 3 비매치: 3PM
  3. (?<=) 는 =뒤에 오는 케릭터 뒤에 있는 경우를 매칭해준다.
  4. (?<!) 는 !뒤에 오는 케릭터가 없는 경우 매칭해준다.

#Flags

  1. global flag는 모든 매치를 찾아준다.
  2. multiline flag는 각 라인별로 따로 찾아준다.
  3. case insensitive 는 대소문자 상관 없이 서치해준다.
  4. .*? 를 통해 lazy matching 첫 finding이 매칭 된다.
